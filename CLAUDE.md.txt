# CLAUDE.md - E-Commerce UI Component Library (Storybook)

## Project Overview

This repository contains the *Storybook component library* for the e-commerce platform migration project. Components built here are published as an npm package (@ecommerce/ui) and consumed by the Next.js application.

### Project Context

- *This Repo*: Storybook component library (ecommerce-ui)
- *Related Repo*: Next.js application (ecommerce-web) - consumes this package
- *Team Size*: 10-15 developers
- *Migration Goal*: Converting .NET e-commerce app to modern JS/TS stack

### Repository Purpose

1. Build reusable, documented UI components
1. Provide visual testing environment via Storybook
1. Publish components as npm package for consumption
1. Maintain design system consistency across applications

### Component Template

typescript
// src/components/primitives/Button/Button.tsx
import { forwardRef, type ButtonHTMLAttributes } from 'react';
import { cn } from '@/utils';

export type ButtonVariant = 'primary' | 'secondary' | 'outline' | 'ghost' | 'destructive';
export type ButtonSize = 'sm' | 'md' | 'lg';

export interface ButtonProps extends ButtonHTMLAttributes<HTMLButtonElement> {
  /** Visual style variant */
  variant?: ButtonVariant;
  /** Button size */
  size?: ButtonSize;
  /** Show loading spinner */
  isLoading?: boolean;
  /** Full width button */
  fullWidth?: boolean;
  /** Left icon */
  leftIcon?: React.ReactNode;
  /** Right icon */
  rightIcon?: React.ReactNode;
}

const variantStyles: Record<ButtonVariant, string> = {
  primary: 'bg-primary-600 text-white hover:bg-primary-700 focus:ring-primary-500',
  secondary: 'bg-secondary-600 text-white hover:bg-secondary-700 focus:ring-secondary-500',
  outline: 'border-2 border-primary-600 text-primary-600 hover:bg-primary-50 focus:ring-primary-500',
  ghost: 'text-primary-600 hover:bg-primary-50 focus:ring-primary-500',
  destructive: 'bg-error text-white hover:bg-error-dark focus:ring-error',
};

const sizeStyles: Record<ButtonSize, string> = {
  sm: 'px-3 py-1.5 text-sm gap-1.5',
  md: 'px-4 py-2 text-base gap-2',
  lg: 'px-6 py-3 text-lg gap-2.5',
};

export const Button = forwardRef<HTMLButtonElement, ButtonProps>(
  (
    {
      variant = 'primary',
      size = 'md',
      isLoading = false,
      fullWidth = false,
      leftIcon,
      rightIcon,
      className,
      disabled,
      children,
      ...props
    },
    ref
  ) => {
    return (
      <button
        ref={ref}
        className={cn(
          // Base styles
          'inline-flex items-center justify-center font-medium rounded-md',
          'transition-colors duration-200',
          'focus:outline-none focus:ring-2 focus:ring-offset-2',
          'disabled:opacity-50 disabled:cursor-not-allowed',
          // Variant & size
          variantStyles[variant],
          sizeStyles[size],
          // Conditional
          fullWidth && 'w-full',
          className
        )}
        disabled={disabled || isLoading}
        {...props}
      >
        {isLoading ? (
          <>
            <LoadingSpinner size={size} />
            <span>Loading...</span>
          </>
        ) : (
          <>
            {leftIcon && <span className="shrink-0">{leftIcon}</span>}
            {children}
            {rightIcon && <span className="shrink-0">{rightIcon}</span>}
          </>
        )}
      </button>
    );
  }
);

Button.displayName = 'Button';

// Internal loading spinner
const LoadingSpinner = ({ size }: { size: ButtonSize }) => {
  const sizeClasses = {
    sm: 'h-3 w-3',
    md: 'h-4 w-4',
    lg: 'h-5 w-5',
  };

  return (
    <svg
      className={cn('animate-spin', sizeClasses[size])}
      fill="none"
      viewBox="0 0 24 24"
    >
      <circle
        className="opacity-25"
        cx="12"
        cy="12"
        r="10"
        stroke="currentColor"
        strokeWidth="4"
      />
      <path
        className="opacity-75"
        fill="currentColor"
        d="M4 12a8 8 0 018-8V0C5.373 0 0 5.373 0 12h4z"
      />
    </svg>
  );
};

### Utility: Class Name Merger (cn)

typescript
// src/utils/cn.ts
import { type ClassValue, clsx } from 'clsx';
import { twMerge } from 'tailwind-merge';

/**
 * Merges Tailwind CSS classes with proper precedence
 * Uses clsx for conditional classes and tailwind-merge for deduplication
 */
export function cn(...inputs: ClassValue[]): string {
  return twMerge(clsx(inputs));
}

## Component Checklist

When creating a new component, ensure:

- [ ] TypeScript interface with JSDoc comments for all props
- [ ] Props have sensible defaults
- [ ] Uses forwardRef for DOM element components
- [ ] Component has displayName set
- [ ] Uses cn() utility for class merging
- [ ] Follows accessibility best practices (ARIA, keyboard navigation)
- [ ] Responsive design considered
- [ ] Storybook story with all variants documented
- [ ] Story has autodocs tag
- [ ] Exported from component’s index.ts
- [ ] Exported from category’s index.ts
- [ ] Exported from main index.ts

## Claude Instructions

### When Working on This Repository

1. *Always check this CLAUDE.md first* for standards
1. *Follow folder structure* exactly as defined
1. *Use TypeScript strict mode* - no any types
1. *Create Storybook stories* for every component
1. *Use forwardRef* for components that wrap DOM elements
1. *Export from all index files* (component → category → root)
1. *Use cn() utility* for all Tailwind class merging

### When Creating Components from Figma/Screenshots

1. Analyze the design for variants, sizes, and states
1. Identify the appropriate category folder
1. Create component with proper TypeScript interfaces
1. Implement all variants shown in design
1. Add comprehensive Storybook stories
1. Ensure accessibility (focus states, ARIA labels)
1. Test responsive behavior if applicable

### Common Patterns

typescript
// Conditional classes
className={cn(
  'base-classes',
  variant === 'primary' && 'primary-classes',
  size === 'lg' && 'large-classes',
  disabled && 'disabled-classes',
  className // Allow override
)}

// ForwardRef pattern
export const Component = forwardRef<HTMLDivElement, ComponentProps>(
  ({ className, ...props }, ref) => {
    return <div ref={ref} className={cn('base', className)} {...props} />;
  }
);
Component.displayName = 'Component';

// Polymorphic component (as prop)
export interface PolymorphicProps<T extends React.ElementType> {
  as?: T;
}
